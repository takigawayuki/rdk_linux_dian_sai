# 激光补偿参数标定指南

本文档给出 `laser_calib.py` 的具体操作步骤，用来标定以下四个参数：

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `REFERENCE_AREA_PIXELS` | 靶板外框在 1500mm 处的像素面积 | 8412 |
| `LASER_X_FIXED` | 激光点在画面中的 X 坐标（固定） | 322 |
| `LASER_Y_MIN_DISTANCE` | 500mm 处激光落点的 Y 坐标 | 240 |
| `LASER_Y_MAX_DISTANCE` | 1500mm 处激光落点的 Y 坐标 | 250 |

标定结果会自动保存到 [laser_calib_params.json](laser_calib_params.json)。

---

## 一、准备工作

### 硬件
- 一张 A4 纸做的靶板（黑色外框矩形，算法靠这个找中心）
- 卷尺或激光测距仪，用来量小车到靶板的真实距离
- 激光器装好，能在墙上/靶板上打出可见光点

### 环境
- 光照稳定（标定过程中别开关灯、别晃相机）
- 靶板正对相机，尽量不要倾斜，否则投影面积会失真
- 相机参数保持和实际运行一致（`my_send2gimbal.py` 里用的 640×480）

### 启动脚本

```bash
cd /home/sunrise/rdk_linux_dian_sai
python3 project1/Test/laser_calib.py
```

画面窗口弹出后，**先用鼠标左键点一下窗口标题栏**，让窗口获得键盘焦点。否则按键会被终端吃掉（`cv2.waitKey` 只认 OpenCV 窗口上的按键）。

按键测试：随便按一个字母，终端应该打印 `[按键] key=xx (x)`。没打印说明焦点还在终端，回去点窗口。

## 二、画面信息说明

标定窗口左上角文字面板：

```
Area  : 8412 px2        ← 当前检测到的靶板轮廓面积（10帧平滑）
Dist  : 1500.0 mm       ← 基于当前参数换算出的距离
LaserY: 250.0 px        ← 按当前参数预测的激光Y坐标
[Params]
REF_AREA=8412
Y_min=240  Y_max=250
X_fixed=322
```

画面中的标记：
- **绿色十字**：画面几何中心（320, 240），参考用
- **黄色十字**：按当前参数预测的激光位置
- **红色十字**：你鼠标点击记录的激光实际落点
- **红色小圆点**：`CenterGet` 检测到的靶板中心

如果 `Area` 一直是 `--` 或 0，说明 `CenterGet` 没识别出靶板。先检查：
- 靶板外框完整在画面里，没被遮挡
- 光照不要太暗
- `project1/Algorithm/CenterGet.py` 里 `threshold_value = 85` 是否适配当前光照（亮环境可能要调高，暗环境调低）

## 三、标定步骤

### 步骤 1：标定 `REFERENCE_AREA_PIXELS`

**目的**：记录靶板在 1500mm 处的像素面积，作为距离反推的基准。

1. 小车静止，相机对着靶板
2. 用卷尺量相机到靶板的距离，调整到 **1500mm**（精度控制在 ±10mm）
3. 观察画面 `Area` 数值，等它稳定（10 帧平滑，晃动一下自然会变）
4. **按 `1`** → 终端会打印 `[标定] 设置 REFERENCE_AREA_PIXELS = XXXX`

参考范围：这个值跟靶板物理大小相关。原来默认 8412（开源代码的 A4 外框）。如果你的靶板尺寸不一样，会得到不同的数字，没关系，记录下来就是了。

**验证**：把小车挪到别的距离（比如 1000mm），看画面 `Dist` 是否接近 1000。偏差超过 50mm 就重新做一次。

---

### 步骤 2：标定 `LASER_Y_MIN_DISTANCE`（500mm 处激光 Y 坐标）

**目的**：记录小车贴近靶板（500mm）时，激光点在画面里的实际 Y 位置。

1. 小车移到距靶板 **500mm**
2. 打开激光器，让激光打在靶板上
3. 确认画面里能看到红色/亮色激光点
4. **鼠标左键点击画面上激光点的位置** → 画面出现红色十字，终端打印 `[标定] 点击坐标: (x, y) → 激光落点 Y = y`
5. **按 `2`** → 终端打印 `[标定] 设置 LASER_Y_MIN_DISTANCE = y`

**点击精度建议**：放大看激光点中心再点，偏 1-2 个像素影响不大。

---

### 步骤 3：标定 `LASER_Y_MAX_DISTANCE`（1500mm 处激光 Y 坐标）

**目的**：记录小车远离靶板（1500mm）时，激光点在画面里的实际 Y 位置。

1. 小车移到距靶板 **1500mm**（就是步骤 1 那个位置）
2. 打开激光器
3. **鼠标左键点击激光点位置**
4. **按 `3`** → 终端打印 `[标定] 设置 LASER_Y_MAX_DISTANCE = y`

**注意**：这里激光 Y 通常比步骤 2 的大（激光器在相机下方时，距离越远激光在画面里越靠下）。如果发现 `Y_max < Y_min`，两个记录顺序可能反了，重新做。

---

### 步骤 4（可选）：标定 `LASER_X_FIXED`

**目的**：如果激光器不是严格竖直对齐相机（X 方向有偏移），把这个值也标了。

1. 小车在任意距离（建议 1000mm，中间位置）
2. 点击激光点
3. **按 `4`** → 终端打印 `[标定] 设置 LASER_X_FIXED = x`

如果激光器对中得很好，激光 X 基本就是 322 左右（画面中心偏右 2px），可以不标。

---

### 步骤 5：验证

做完 1-3 后，小车来回移动，观察：

- **黄色十字（预测激光位置）** 应该跟 **实际激光点** 大致重合
- 500mm 处和 1500mm 处，预测 Y 应该分别等于你标定的值
- 中间距离（如 1000mm）预测 Y 应该是两端的平均（线性插值）

如果预测位置和实际激光位置偏差稳定在 5px 以内，标定就够用了。偏差大就重新做对应的步骤。

## 五、按键速查表

| 按键 | 动作 |
|------|------|
| 鼠标左键 | 在画面上标记激光落点 |
| `1` | 记录当前平滑面积为 `REFERENCE_AREA_PIXELS`（1500mm 处按） |
| `2` | 记录点击点 Y 为 `LASER_Y_MIN_DISTANCE`（500mm 处按） |
| `3` | 记录点击点 Y 为 `LASER_Y_MAX_DISTANCE`（1500mm 处按） |
| `4` | 记录点击点 X 为 `LASER_X_FIXED` |
| `r` | 清除鼠标点击标记 |
| `s` | 保存当前帧到 `/tmp/calib_<timestamp>.jpg`（留档用） |
| `q` | 退出，打印最终参数 |

## 六、常见问题

**Q：按 `1/2/3/4` 没反应？**
A：焦点不在 OpenCV 窗口上。鼠标点一下 `laser_calib` 窗口让它获得焦点，再按。随便按一个键看终端有没有 `[按键] key=XX`，有就说明焦点对了。

**Q：Area 一直是 0？**
A：`CenterGet` 没识别到矩形。检查：靶板完整在画面里 / 光照够 / `threshold_value` 是否合适。必要时打开 OpenCV 的二值化调试看看。

**Q：激光点看不见？**
A：激光太弱被环境光淹没了，或者 MJPG 压缩把高亮像素削掉了。可以关灯试试；或者在靶板上贴白纸增强反射。

**Q：预测位置总是跟实际差一截？**
A：可能是重新测距不准。重新做一遍步骤 1-3，注意每次距离测量都要准确。也可能是激光器装歪了，Y 坐标不是严格线性的，这时候考虑加中间点（多点标定、分段插值）。

**Q：标定完了怎么删除旧参数重来？**
A：删除 `project1/Test/laser_calib_params.json`，重启脚本就是默认值。

## 七、后续步骤

标定完成后，下一步是在 `my_send2gimbal.py` 里集成激光补偿逻辑：

1. `CenterGet` 已经支持 `return_area=True`，可以直接拿面积
2. 面积 → 距离：`D = 1500 × √(REFERENCE_AREA_PIXELS / area)`
3. 距离 → 激光 Y：线性插值（公式见 `激光视差补偿原理.md`）
4. 发送给云台的误差改为：`dy = laser_y - target_center_y`（激光点相对靶心的误差，而不是靶心相对画面中心）
